About

프로젝트 소개

What is SDACS?

하늘의 신호등 — 드론이 스스로 만드는 이동형 가상 레이더 돔

Traffic lights in the sky — drones become the radar.

"레이더를 땅에 설치하는 대신, 드론 자체가 레이더가 되면 어떨까?"
"What if the drones themselves became the radar?"

SDACS는 이 단순한 발상에서 출발했습니다. 20대의 관제 드론이 공중에 올라가 그물망처럼 연결된 감시 체계 (이동형 가상 레이더 돔)를 스스로 만들어, 도심 하늘을 자동으로 감시하고 충돌을 미리 막는 시스템입니다.

SDACS starts from that idea. Twenty control drones take off and self-organize into a networked surveillance mesh — a mobile virtual radar dome — that monitors the urban sky and prevents collisions before they happen.

지금 이 순간에도 전국에서 90만 대 이상의 드론이 같은 낮은 하늘(지상 120m 이하)을 공유하고 있습니다. 도심 저고도의 67%는 기존 레이더의 사각지대이고, 수동 관제의 평균 반응시간은 5분입니다. SDACS는 이 문제를 분산형 APF 충돌 회피와 CPA 기반 90초 선제 예측으로 해결합니다.

Today, 900,000+ drones share the low-altitude layer below 120m AGL. 67% of that low-altitude urban airspace is a ground-radar blind spot, and manual ATC typically needs ~5 minutes to react. SDACS solves this with distributed APF-based avoidance and CPA-based 90-second look-ahead.

SimPy 이산 이벤트 엔진 위에서 2,668개의 자동화 테스트와 38,400회 Monte Carlo 검증을 거쳤으며, 7대 광역시 도시환경 63개 시나리오에서 충돌 해결률 97.8% 이상, 응답 지연 1초 미만을 달성했습니다.

Built on a SimPy discrete-event engine, SDACS is backed by 2,668 automated tests and 38,400 Monte Carlo runs. Across 63 scenarios spanning seven Korean metropolitan areas, it achieves >97.8% conflict resolution with sub-1-second response latency.

Features

핵심 기능

Core Features

SDACS를 구성하는 여섯 개의 기둥

Six pillars that make SDACS work.

🧭

분산형 APF 회피

Distributed APF Avoidance

인공 포텐셜 필드로 드론마다 인력(목적지) + 척력(타 기체·지형)을 10Hz로 계산해 개별 회피

Per-drone attractive/repulsive fields computed at 10Hz for fully decentralized avoidance.

⏱️

CPA 90초 선제 예측

90s CPA Look-Ahead

Closest Point of Approach 기반 90초 앞 충돌 위험을 미리 찾고, RA 6종 중 적합한 것을 발행

Closest Point of Approach predicts conflicts 90s ahead and triggers one of six advisories.

🗺️

Voronoi 공역 분할

Voronoi Partitioning

AirspaceController가 동적 Voronoi 분할로 책임 공역을 배정 — 충돌 영역 감소

Dynamic Voronoi cells assign responsibility zones and shrink conflict surfaces.

🌬️

바람 적응 제어

Weather-Adaptive Control

WindModel(난류 + 평균풍)을 APF 척력에 주입해 강풍·돌풍 상황에서도 안정적인 경로

Gust + mean-wind model is injected into the APF repulsion, keeping trajectories stable.

🧠

ML 충돌 예측 (옵션)

ML Collision Prediction

PyTorch FP16 추론 + PPO 강화학습 회피 정책(ONNX 내보내기) — 고밀도 시나리오 보강

FP16 PyTorch inference + PPO RL policy (ONNX exportable) boost high-density scenarios.

🖥️

WebGPU 3D 시각화

WebGPU 3D Visualization

브라우저만 있으면 돌아가는 Three.js + WGSL 시뮬레이터 — 설치 없이 즉시 검증

Three.js + WGSL simulator runs in any browser — no install, instant validation.

Architecture

3-Layer + UI 아키텍처

3-Layer + UI Architecture

드론 에이전트 · 공역 관제 · 시뮬레이션 엔진 · 사용자 인터페이스

Drone agent · airspace control · simulation engine · user interface

Layer 1

드론 에이전트

Drone Agent

10Hz SimPy 프로세스. 위치·속도·배터리 갱신, FSM 전이 (Idle → Takeoff → Cruise → Avoid → Landing)

10Hz SimPy process. State updates + FSM (Idle → Takeoff → Cruise → Avoid → Landing).

Layer 2

공역 관제

Airspace Control

1Hz AirspaceController. CPA 기반 90초 예측, Voronoi 분할, 6종 Resolution Advisory 자동 발행

1Hz controller. CPA look-ahead, Voronoi partitioning, auto-issues 6 advisories.

Layer 3

시뮬레이션 엔진

Simulation Engine

SwarmSimulator + WindModel + Monte Carlo. 384 config × 100 seeds = 38,400 run 검증

SwarmSimulator + WindModel + MC. 384 configs × 100 seeds = 38,400 runs.

Layer 4

사용자 인터페이스

User Interface

CLI (main.py) + Dash 3D 시각화 + Three.js WebGPU 웹 시뮬레이터

CLI (main.py) + Dash 3D dashboard + Three.js WebGPU web simulator.
