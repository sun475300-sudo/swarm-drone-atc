Fig. 3 — 분산 CD&R 메시지 시퀀스 (Sequence Diagram)

sequenceDiagram
  participant EXT as 외부 이벤트
  participant CTRL as 공역 제어기
  participant STORE as 공역 상태 저장소
  participant D1 as 드론 #1
  participant D2 as 드론 #2
  participant DN as 드론 #N
  participant DOUT as 드론 #X (통신단절)

  EXT ->> CTRL: 유인기 진입 이벤트
  CTRL ->> CTRL: 펜스 형상/크기 결정
  CTRL ->> STORE: fence.create(r=500m, TTL=600s)
  CTRL -) D1: broadcast(fence_payload)
  CTRL -) D2: broadcast(fence_payload)
  CTRL -) DN: broadcast(fence_payload)
  CTRL -x DOUT: 브로드캐스트 실패
  par 각 드론 병렬 재계산
    D1 ->> D1: apf.recompute(...)
    D2 ->> D2: apf.recompute(...)
    DN ->> DN: apf.recompute(...)
  end
  D1 -->> CTRL: ack(new_path)
  D2 -->> CTRL: ack(new_path)
  DN -->> CTRL: ack(new_path)
  note over DOUT: 로컬 캐시 기반 폴백
→ 기본 회피 유지
  DOUT ->> DOUT: local_cache.recompute()

Fig. 4 — APF 벡터 합성 · 강풍 모드 전환 (FSM)

stateDiagram-v2
  [*] --> DEFAULT
  state DEFAULT {
    direction LR
    d_default: k_att=1.0
k_rep=2.0
d_safe=30m
v_max=15 m/s
  }
  state WINDY {
    direction LR
    d_windy: k_att=0.6
k_rep=3.5
d_safe=50m
v_max=10 m/s
  }
  DEFAULT --> WINDY: 풍속 > 10 m/s (임계)
  WINDY --> DEFAULT: 풍속 ≤ 10 m/s (hysteresis 2 m/s)
  DEFAULT --> DEFAULT: F = F_attr + ΣF_rep
  WINDY --> WINDY: F = F_attr + ΣF_rep\n(robust gains)
