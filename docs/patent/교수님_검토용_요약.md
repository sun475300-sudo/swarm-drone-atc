## 3. 핵심 기술 요지 (5분 브리핑용)

### 3-1. 해결하고자 하는 문제

현행 드론 공역 관제는 (i) 정적 지오펜스, (ii) 중앙집중형 관제(단일 실패점, SPOF), (iii) 대규모 군집의 실시간 연산 한계, (iv) 기상 적응성 부재라는 4가지 기술적 한계를 가집니다.

### 3-2. 본 발명의 4가지 구성 요소

1. **동적 3차원 지오펜싱** — 이벤트 기반 원통·구체 가상 펜스 실시간 생성 및 TTL 기반 자동 제거
2. **분산 충돌 탐지·회피 (CD&R)** — 중앙 지시 없이 드론 단위 자율 경로 재계산 → SPOF 원천 제거
3. **APF 기반 경로 계획** — 인력·반발력 벡터 합, **강풍 모드 자동 전환** (풍속 10 m/s 임계)
4. **WebGPU Compute 병렬 가속** — APF·CBS·Voronoi 연산의 GPU 병렬화로 O(N²) → 사실상 O(log N)

### 3-3. 이 네 요소의 **융합 구성**이 청구항 1(독립항)의 핵심이며, 종래 기술 대비 차별성의 근거입니다.

## 4. 실증 성과 (정량 지표)

| 지표 | 측정값 | 검증 방법 |
|------|--------|-----------|
| 충돌 해결률 | **97.8%** (978/1,000) | 1,000회 반복 실증 |
| 평균 응답 시간 | **< 1.2 초** | 위험 탐지 → 경로 재산출 |
| WebGPU 가속비 | CPU 대비 **1.05배 이상** | 엣지 GPU 기준 |
| 강풍 모드 안정성 | **≥ 97%** 해결률 | 풍속 10 m/s 환경 |
| Monte Carlo 스윕 | **38,400회** 전수 통과 | 파라미터 공간 탐색 |
| 시나리오 검증 | **63개** (7 광역시 × 9 시나리오) | 전수 통과 |
| 자동화 테스트 | **1,841개** 이상 | pytest 프레임워크 |

**공식**: 충돌 해결률 = 1 − collisions / (conflicts + collisions)

## 6. 교수님께 요청드리는 검토 포인트

> 다음 7개 항목에 대한 의견을 본 문서 하단 검토 의견란, 또는 본 명세서 표지의 의견란에 기재 부탁드립니다.

1. **기술분야의 적정성**: IPC G08G 5/00 · G05D 1/10 · B64C 39/02의 선택이 타당한지
2. **청구항 1(독립항)의 구성요소**: 4개 모듈의 결합 구성이 **단일 발명 개념**으로 성립하는지
3. **종속항의 종속 관계**: 청구항 2~5의 종속 구조가 권리범위를 적절히 축소하는지
4. **수치 한정의 적정성**:
   - "풍속 10 m/s" 임계치가 실시예에 머물러야 하는지, 종속항에 유지해야 하는지
   - "97.8%", "38,400회" 등의 실험 수치를 청구항에 포함시키지 않은 것이 옳은지
5. **선행기술 대비 진보성**: NASA UTM, Airbus UTM, Wing, Amazon Prime Air 등과의 차별 포인트 강조 방안
6. **도면 7매의 기술 표현**: 도 3(분산 CD&R 시퀀스)이 신규 작성 필요 — 시퀀스 다이어그램 구성 방안 제안 요청
7. **연구 윤리 및 학술 기여도**: 본 발명을 캡스톤디자인 보고서 기반으로 출원할 경우, **학과 지적재산권 규정** 상 공동 발명자 · 권리 귀속 관련 확인
