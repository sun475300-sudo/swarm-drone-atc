## 2. 특허 출원용 도면 작성 규칙 (KIPO 기준)

1. **색상 금지**: 원칙적으로 흑백 선화(line drawing). 회색 음영은 최소화.
2. **참조부호(도면부호)**: 모든 구성요소에 숫자 부호 부여 (예: `100` 드론 에이전트, `200` 공역 제어기, `210` 동적 지오펜싱 모듈).
3. **해상도**: TIFF 300dpi 이상 또는 SVG/PDF 벡터.
4. **글꼴**: 한글/영문 명확히 식별 가능한 크기(최소 3pt 이상).
5. **사진 사용 금지**: 실제 드론 사진이 아닌 개념도/블록도만 허용.
6. **A4 단면**: 각 도면은 A4 용지 한 면에 수록되도록 배치.

## 4. 도면별 상세 사양

### 【도 1】 4계층 시스템 아키텍처
- **구성요소**: Layer 1 (100) ↔ Layer 2 (200) ↔ Layer 3 (300) ↔ Layer 4 (400)
- **연결선 표기**: 10Hz/1Hz 주기 및 데이터 흐름 방향 명시
- **기본 원본**: `docs/images/architecture.svg`

### 【도 2】 동적 지오펜싱 절차 흐름도
- **단계**: S210 이벤트 감시 → S220 펜스 형상·크기 결정 → S230 저장소 기록(TTL) → S240 APF 반발 소스 주입 → S250 TTL 만료 자동 제거
- **기본 원본**: `docs/images/algorithm_flow.svg` (단계명 교체 필요)

### 【도 3】 분산 CD&R 시퀀스 (신규)
- **액터**: 공역 제어기(200), 드론 A(100-A), 드론 B(100-B), 드론 C(100-C)
- **시퀀스**: 펜스 생성 → 브로드캐스트 → 각 드론 독립 재계산 → 로컬 캐시 업데이트
- **작성 도구**: Mermaid sequenceDiagram 권장

### 【도 4】 APF 벡터 합성 및 강풍 모드
- **좌측 서브뷰**: 기본 모드 벡터 합성 (F_attr + ΣF_rep)
- **우측 서브뷰**: 강풍 모드 (k_rep 상향, k_att 하향)
- **전환 조건**: 풍속 센서 입력 > 10 m/s

### 【도 5】 WebGPU Compute 파이프라인
- **단계**: CPU 위치 버퍼 → GPU 업로드(510) → Workgroup 디스패치(520) → APF 병렬 계산 → 결과 다운로드 → 제어 명령 변환
- **강조점**: Coalesced memory access, O(N²) → O(log N)급 성능

### 【도 6】 Monte Carlo 통계 분포
- **축**: X = 충돌 해결률(%), Y = 누적 확률
- **데이터**: 38,400회 스윕 결과
- **참고 원본**: `docs/images/conflict_resolution_heatmap.png` (CDF로 재가공 권장)

### 【도 7】 7개 광역시 KPI 레이더 차트
- **축**: 충돌해결률, 평균응답시간, 처리량, 전력소모, 강풍안정성 등 5~7축
- **도시**: 서울·부산·대구·인천·광주·대전·울산
- **기본 원본**: `docs/images/scenario_kpi_radar.png`
